CSS Units
---------

CSS units define **lengths, sizes, and spacing**. They can be **absolute** (fixed values) or **relative** (depend on parent, root, or viewport).

### Absolute Units

Fixed sizes that do not change with screen size.

*   `px` – pixels (commonly used)
*   `pt` – points (print)
*   `cm`, `mm`, `in` – rarely used for screen

    h1 { font-size: 32px; }
    p { font-size: 12pt; }

### Relative to Font Size

Scale relative to the font size of parent (`em`) or root (`rem`).

    .parent { font-size: 16px; }
    .child { font-size: 1.5em; } /* 24px */
    .rem-text { font-size: 2rem; } /* relative to root */

### Percentages

Percentages are relative to a parent property (like width or font-size).

    .box { width: 50%; background: lightblue; }
    p { font-size: 120%; }

Font size 120%

### Viewport Units

Relative to the browser window size.

*   `vw` – % of viewport width
*   `vh` – % of viewport height
*   `vmin`, `vmax` – smaller or larger of `vw` and `vh`

    h1 { font-size: 5vw; }
    .full-height { height: 50vh; }

Boxes & the Box Model
---------------------

Every element in CSS is a **box**. The _box model_ describes how width, height, padding, border, and margin work together to determine the final size and spacing of an element.

### Box Structure

The box model has four main layers:

*   **Content** – text or child elements inside
*   **Padding** – space between content and border
*   **Border** – line surrounding padding
*   **Margin** – space outside the element

    .box {
      width: 200px;
      height: 100px;
      padding: 20px;
      border: 5px solid #3b82f6;
      margin: 20px;
    }

Responsive Design
-----------------

Responsive design ensures that a website looks and works well on all screen sizes, from mobile to desktop.

### Viewport

Always include the viewport meta tag to control scaling on mobile devices:

    <meta name="viewport" content="width=device-width, initial-scale=1">

This allows the layout to respond to the actual screen width.
