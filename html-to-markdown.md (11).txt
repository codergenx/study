Client-Side Routing
-------------------

Frontend routing allows Single Page Applications (SPAs) to change views without reloading the page.

It improves user experience by making navigation seamless, faster, and more app-like.

Instead of requesting a new HTML page from the server, routing swaps components on the client side.

### Traditional vs SPA Routing

#### Traditional Multi-Page App

    User clicks link
      ↓
    Browser requests /about
      ↓
    Server sends new HTML
      ↓
    Browser loads entire page
      ↓
    Screen flashes/reloads

Full page reload every time

#### SPA with Client-Side Routing

    User clicks link
      ↓
    JavaScript intercepts
      ↓
    URL changes (no request!)
      ↓
    React swaps component
      ↓
    Smooth transition

No page reload, instant

### React Router

React Router is the standard routing library for React applications.

Routes are mapped to React components, so different URLs render different components without refreshing.

It supports dynamic routes, nested routes, programmatic navigation, and protected routes.

### Installing React Router

    npm install react-router-dom

Current version: v6 (major changes from v5)

We'll use React Router v6 syntax

### Fetching Data with Route Params

    import { useParams } from 'react-router-dom';
    import { useState, useEffect } from 'react';
    import api from '../api/axios';
    
    function StudentDetail() {
      const { id } = useParams();
      const [student, setStudent] = useState(null);
      const [loading, setLoading] = useState(true);
    
      useEffect(() => {
        const fetchStudent = async () => {
          try {
            const response = await api.get(`/students/${id}`);
            setStudent(response.data.data);
            setLoading(false);
          } catch (error) {
            console.error('Error:', error);
            setLoading(false);
          }
        };
        
        fetchStudent();
      }, [id]);  // Refetch when ID changes
    
      if (loading) return <div>Loading...</div>;
      if (!student) return <div>Student not found</div>;
    
      return (
        <div>
          <h1>{student.name}</h1>
          <p>Email: {student.email}</p>
        </div>
      );
    }

### Programmatic Navigation

    import { useNavigate } from 'react-router-dom';
    
    function LoginForm() {
      const navigate = useNavigate();
    
      const handleLogin = async (e) => {
        e.preventDefault();
        // Login logic...
        
        // Navigate to dashboard after successful login
        navigate('/dashboard');
        
        // Navigate with replace (can't go back)
        navigate('/dashboard', { replace: true });
        
        // Go back
        navigate(-1);
        
        // Go forward
        navigate(1);
        
        // Navigate with state
        navigate('/profile', { state: { from: 'login' } });
      };
    
      return (
        <form onSubmit={handleLogin}>
          <button type="submit">Login</button>
        </form>
      );
    }

### Protected Routes (Authentication)

    function ProtectedRoute({ children }) {
      const isAuthenticated = localStorage.getItem('token');
      
      if (!isAuthenticated) {
        return <Navigate to="/login" replace />;
      }
      
      return children;
    }
    
    // Usage in App.jsx
    function App() {

State Management
----------------

As React apps grow, managing shared state across components becomes challenging.

### The Props Drilling Problem

    function App() {
      const [user, setUser] = useState({ name: 'Alice', role: 'Admin' });
      
      return <Layout user={user} setUser={setUser} />;
    }
    
    function Layout({ user, setUser }) {
      return <Sidebar user={user} setUser={setUser} />;
    }
    
    function Sidebar({ user, setUser }) {
      return <UserMenu user={user} setUser={setUser} />;
    }
    
    function UserMenu({ user, setUser }) {
      return <div>Welcome, {user.name}!</div>;
    }
    
    // UserMenu needs user, but we passed through 3 components!
    // This is "Props Drilling" - messy and hard to maintain
    

Props drilling makes code hard to maintain and refactor

### Visualization: Props Drilling

App (has user state)
  ↓ (passes user as prop)
Layout (doesn't need user, just passes it)
  ↓ (passes user as prop)
Sidebar (doesn't need user, just passes it)
  ↓ (passes user as prop)
Navigation (doesn't need user, just passes it)
  ↓ (passes user as prop)
UserMenu (finally uses user!)

Problem: Intermediate components are just "prop couriers"
						

### Solution: Context API

React Context provides a way to share data across the component tree without passing props manually at every level.

It creates a "global" state that any component can access directly.

Perfect for: user authentication, theme, language, shopping cart, etc.

### Creating a Context - Basic Example

`context/UserContext.jsx`

    import { createContext, useContext, useState } from 'react';
    
    // 1. Create Context
    const UserContext = createContext();
    
    // 2. Create Provider Component
    export function UserProvider({ children }) {
      const [user, setUser] = useState(null);
    
      return (
        <UserContext.Provider value={{ user, setUser }}>
          {children}
        </UserContext.Provider>
      );
    }
    
    // 3. Create custom hook for easy access
    export function useUser() {
      const context = useContext(UserContext);
      if (!context) {
        throw new Error('useUser must be used within UserProvider');
      }
      return context;
    }

### Using the Context

    // Wrap app with Provider
    import { UserProvider } from './context/UserContext';
    
    function App() {
      return (
        <UserProvider>
          <Layout />
        </UserProvider>
      );
    }
    
    // Access context anywhere in the tree
    import { useUser } from './context/UserContext';
    
    function UserMenu() {
      const { user, setUser } = useUser();
      
      return (
        <div>
          {user ? (
            <p>Welcome, {user.name}!</p>
          ) : (
            <button onClick={() => setUser({ name: 'Alice' })}>
              Login
            </button>
          )}
        </div>
      );
    }
    
    // No props drilling! Direct access from any component
    

### Complete Context Example - Auth

`context/AuthContext.jsx`

    import { createContext, useContext, useState } from 'react';
    
    const AuthContext = createContext();
    
    export function AuthProvider({ children }) {
      const [user, setUser] = useState(null);
    
      const login = (username, password) => {
        // Simplified login
        if (username && password) {
          setUser({ username, id: 1 });
          localStorage.setItem('token', 'fake-jwt-token');
          return true;
        }
        return false;
      };
    
      const logout = () => {
        setUser(null);
        localStorage.removeItem('token');
      };
    
      const isAuthenticated = () => {
        return user !== null;
      };
    
      return (
        <AuthContext.Provider value={{ user, login, logout, isAuthenticated }}>
          {children}
        </AuthContext.Provider>
      );
    }
    
    export function useAuth() {
      const context = useContext(AuthContext);
      if (!context) {
        throw new Error('useAuth must be used within AuthProvider');
      }
      return context;
    }

    import { useAuth } from '../context/AuthContext';
    import { useNavigate } from 'react-router-dom';
    
    function Login() {
      const { login } = useAuth();
      const navigate = useNavigate();
      const [form, setForm] = useState({ username: '', password: '' });
    
      const handleSubmit = (e) => {
        e.preventDefault();
        if (login(form.username, form.password)) {
          navigate('/dashboard');
        } else {
          alert('Login failed');
        }
      };
    
      return (
        <form onSubmit={handleSubmit}>
          <input value={form.username} onChange={(e) => setForm({...form, username: e.target.value})} />
          <input type="password" value={form.password} onChange={(e) => setForm({...form, password: e.target.value})} />
          <button type="submit">Login</button>
        </form>
      );
    }

### Theme Context Example

    import { createContext, useContext, useState } from 'react';
    
    const ThemeContext = createContext();
    
    export function ThemeProvider({ children }) {
      const [theme, setTheme] = useState('light');
    
      const toggleTheme = () => {
        setTheme(prev => prev === 'light' ? 'dark' : 'light');
      };
    
      return (
        <ThemeContext.Provider value={{ theme, toggleTheme }}>
          {children}
        </ThemeContext.Provider>
      );
    }
    
    export const useTheme = () => useContext(ThemeContext);
    
    // Usage
    function ThemeToggle() {
      const { theme, toggleTheme } = useTheme();
      
      return (
        <button onClick={toggleTheme}>
          Current: {theme}
        </button>
      );
    }

### Context Best Practices

*   Create separate contexts for different concerns (auth, theme, cart)
*   Always create a custom hook for accessing context (`useAuth`, `useTheme`)
*   Place Provider as high as needed, but not higher
*   Don't overuse Context - local state is fine for component-specific data
*   Use localStorage/sessionStorage to persist important data
*   Consider performance - Context causes re-renders of all consumers

### When to Use Context vs Props

Use Props When...

Use Context When...

Parent-child relationship

Many components need same data

Only 1-2 levels deep

Props drilling through 3+ levels

Component reusability important

Truly global app state (auth, theme)

Data specific to one component

Data needed across routes

Performance critical updates

Infrequent updates (user login/logout)

### Complete Example - Shopping Cart Context

`context/CartContext.jsx`

    import { createContext, useContext, useState } from 'react';
    
    const CartContext = createContext();
    
    export function CartProvider({ children }) {
      const [items, setItems] = useState([]);
    
      const addToCart = (product) => {
        setItems(prev => {
          const existing = prev.find(item => item.id === product.id);
          if (existing) {
            return prev.map(item =>
              item.id === product.id
                ? { ...item, quantity: item.quantity + 1 }
                : item
            );
          }
          return [...prev, { ...product, quantity: 1 }];
        });
      };
    
      const removeFromCart = (productId) => {
        setItems(prev => prev.filter(item => item.id !== productId));
      };
    
      const updateQuantity = (productId, quantity) => {
        setItems(prev =>
          prev.map(item =>
            item.id === productId ? { ...item, quantity } : item
          )
        );
      };
    
      const getTotalPrice = () => {
        return items.reduce((total, item) => total + item.price * item.quantity, 0);
      };
    
      const getTotalItems = () => {
        return items.reduce((total, item) => total + item.quantity, 0);
      };
    
      return (
        <CartContext.Provider value={{
          items,
          addToCart,
          removeFromCart,
          updateQuantity,
          getTotalPrice,
          getTotalItems
        }}>
          {children}
        </CartContext.Provider>
      );
    }
    
    export const useCart = () => useContext(CartContext);
