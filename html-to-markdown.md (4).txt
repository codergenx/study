### Challenges with Vanilla CSS

*   Global namespace & selector collisions (specificity wars)
*   Large maintenance burden as project scales
*   Inconsistent design decisions across teams
*   Difficulty composing small utility rules; repeated declarations

### CSS Frameworks to the rescue

Frameworks provide a consistent, tested set of styles and utilities that speed up development, reduce duplication, and enforce design patterns.

### How frameworks solve Vanilla CSS problems

*   Namespacing and predictable classes reduce collisions
*   Reusable components and utilities reduce repetition
*   Opinionated defaults provide consistency
*   Integrated tooling improves workflow (build, purge, watch)

### Some popular CSS Frameworks

### Component-first

*   Bootstrap
*   Bulma

### Utility-first

*   Tailwind CSS

### What is Tailwind CSS?

Tailwind is a utility-first CSS framework — it offers tiny, single-purpose utility classes (e.g. p-4, text-center) that you compose to build UI directly in your markup.

*   Not a component library — it's a toolbox
*   Highly configurable via a JavaScript config (tailwind.config.js)
*   Pairs well with build tooling to purge unused CSS

### Easiest way to try: CDN

You can include Tailwind via a CDN for quick prototypes and slides. This skips the build step and gives you access to utility classes right away.

Hello Tailwind

    <!-- Tailwind CDN (for demos) -->
    <link href="https://cdn.jsdelivr.net/npm/tailwindcss@3.4.2/dist/tailwind.min.css" rel="stylesheet">
    
    <!-- then use utilities in your markup -->
    <div class="p-6 bg-slate-500 text-white rounded">Hello Tailwind</div>
    

### CDN — Not intended for production

Use the npm + build setup for production so you can:

*   Purge unused classes (small CSS footprint)
*   Extend/override tokens in tailwind.config.js
*   Enable JIT mode and advanced plugins

### Tailwind — Core utilities (overview)

We'll show short examples for colors, spacing, layout, typography, and interactivity.

### Colors

Tailwind exposes a palette with shades. Use classes like bg-blue-500, text-slate-700, or custom colors from your config.

*   Background: bg-{color}-{shade}
*   Text: text-{color}-{shade}
*   Border: border-{color}-{shade}

Blue to Indigo banner

Paragraph with red text

    <div class="p-4 bg-gradient-to-r from-blue-400 to-indigo-600 text-white rounded">
      Blue to Indigo banner
    </div>
    
    <p class="text-red-500 mt-4">Paragraph with slate text</p>
    

### Margin & Padding

Spacing utilities follow a scale (e.g. 0, 0.5, 1, 1.5, 2, 3, 4, 5, 6 ... usually mapped to 0, 0.125rem, 0.25rem, 0.375rem, 0.5rem, 0.75rem, 1rem etc.).

Card with spacing

Inline padded area

    <div class="p-6 mb-16 bg-red-200">Card with spacing</div>
    
    <div class="px-16 py-2 bg-red-200">Inline padded area</div>

Use helpers like w-1/2, w-full, h-64 or arbitrary values like w-\[320px\].

### Borders & Shadows

Controls for border width, color, radius and shadows are provided with concise classes like border, rounded-lg, shadow-md.

    <div class="border border-gray-200 rounded-lg shadow p-4">Card</div>
    
    <div class="rounded-full border-2 border-indigo-500 p-2 inline-block">Avatar placeholder</div>
    

### Text sizes & font weights

Use text-sm, text-xl, and weights like font-semibold, font-bold.

    <h3>Heading</h3>
    <p class="text-sm text-slate-500">Small helper text</p>
    

### Position

Helpers: relative, absolute, inset-0, top-1/2.

    <div class="relative h-40 bg-slate-50">
      <div class="absolute inset-0 flex items-center justify-center">Centered overlay</div>
    </div>
    

### Visibility & Z-index

Classes: hidden, block, z-10, z-50.

    <div class="relative">
      <div class="z-10 relative">Foreground</div>
      <div class="absolute inset-0 z-0 opacity-30">Background layer</div>
    </div>
    

### Flex layout

Utilities for flexbox: flex, items-center, justify-between, gap-4.

    <div class="flex items-center justify-between p-4 bg-white rounded">
      <div>Left</div>
      <div>Right</div>
    </div>
    

### Grid layout

Grid utilities: grid, grid-cols-3, col-span-2.

    <div class="grid grid-cols-3 gap-4">
      <div class="col-span-2 p-4 bg-white rounded">Main</div>
      <div class="p-4 bg-white rounded">Aside</div>
    </div>
    

### Transition & Animation

Use transition, duration-200, ease-in-out. For animations, leverage @keyframes via custom CSS or plugins.

Hover me

### Transform

Helpers: scale-95, rotate-3, translate-x-2. Often used with transform and transitions.

Interactive tile

    <div class="transform hover:scale-110 duration-700">Interactive tile</div>
    

### Introduction to npm

npm (Node Package Manager) lets you install, version and run packages and build tools — required for production Tailwind workflows (postcss, autoprefixer, tailwindcss).

### Install Tailwind via npm (typical setup)

1.  Initialize project: npm init -y
2.  Install: npm install -D tailwindcss postcss autoprefixer
3.  Generate config: npx tailwindcss init -p
4.  Configure content paths and build script to produce your CSS.

    // package.json scripts snippet
    "scripts": {
      "build:css": "tailwindcss -i ./src/input.css -o ./dist/output.css --minify",
      "watch:css": "tailwindcss -i ./src/input.css -o ./dist/output.css --watch"
    }
    
    /* input.css */
    @tailwind base;
    @tailwind components;
    @tailwind utilities;
    

### Extending Tailwind

Use tailwind.config.js to add custom tokens, fonts, sizes and enable plugins.

### Define new colors

Add colors under the theme.extend.colors key and then use them like bg-mybrand-500.

    // tailwind.config.js
    module.exports = {
      content: ['./src/**/*.{html,js}'],
      theme: {
        extend: {
          colors: {
            mybrand: {
              50: '#f5f7ff',
              500: '#3b82f6',
              700: '#1e40af',
            }
          }
        }
      }
    }
    

### Adding new fonts

Extend theme.extend.fontFamily and reference via font-sans or your custom name.

    // tailwind.config.js
    module.exports = {
      theme: {
        extend: {
          fontFamily: {
            inter: ['Inter', 'ui-sans-serif', 'system-ui']
          }
        }
      }
    }
    
    /* then in CSS */
    <body class="font-inter">...</body>
    

### Redefining sizes

Extend spacing, fontSize or container sizes in config to support project-specific design tokens.

    // tailwind.config.js
    module.exports = {
      theme: {
        extend: {
          spacing: {
            '72': '18rem',
            '84': '21rem'
          },
          fontSize: {
            'xxs': '0.625rem'
          }
        }
      }
    }
    

### Watching changes

Use the \--watch flag during development so Tailwind recompiles CSS on file changes.

    // Run during development
    npm run watch:css
    
    // or directly
    npx tailwindcss -i ./src/input.css -o ./dist/output.css --watch
