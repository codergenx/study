### What is an ORM?

ORM = **Object Relational Mapper**

It maps database tables to language-native objects (classes) so you can work with your data using code instead of writing SQL for common operations.

*   Simplifies CRUD (Create, Read, Update, Delete)
*   Helps avoid repetitive SQL
*   Makes code more expressive and testable
*   Provides abstraction over different database engines

### Raw SQL vs ORM - Comparison

#### Raw SQL

    -- Get active users
    SELECT * FROM users 
    WHERE active = true 
    ORDER BY created_at DESC 
    LIMIT 10;

❌ String-based, error-prone, no autocomplete

#### Prisma ORM

    const users = await prisma.user.findMany({
      where: { active: true },
      orderBy: { createdAt: 'desc' },
      take: 10
    });

✅ Type-safe, autocomplete, refactor-friendly

Prisma reads like JavaScript — easier to refactor and test. Behind the scenes it still issues SQL to the DB.

### Why Use an ORM?

Benefit

Explanation

**Type Safety**

Catch errors at compile time, not runtime

**Productivity**

Write less boilerplate code

**Maintainability**

Easier to refactor and understand

**Database Agnostic**

Switch databases with minimal code changes

**Relationships**

Easy to work with related data

**Migrations**

Track and version database schema changes

### Database Abstraction

ORMs provide a database-agnostic layer

Your queries are written in JavaScript/TypeScript and the ORM translates them to SQL specific to the configured database (PostgreSQL, MySQL, SQLite, etc.)

If you switch database engines, **most** ORM code keeps working — that is the power of abstraction.

    // Same Prisma code works with PostgreSQL, MySQL, SQLite, SQL Server
    const user = await prisma.user.findUnique({ where: { id: 1 } });
    
    // Prisma generates the right SQL for each database automatically

### Can an ORM Do Everything?

Short answer: **NO**

ORMs work for most common use cases including selecting, filtering, joining, ordering, and aggregations.

But sometimes, you need to perform operations that the database can handle much faster with raw SQL.

**Best Practice:** Prefer ORM for maintainability; use raw SQL for complex queries or micro-optimizations.

### When to Use Raw SQL

Use raw SQL when Prisma cannot express the query

**Query**

    // Ranking users with a window function
            const result = await prisma.$queryRaw`
              SELECT 
                id, 
                name, 
                score,
                RANK() OVER (ORDER BY score DESC) as rank
              FROM users
              WHERE department = 'Engineering'
            `;
            

**Sample Output**

    
            id | name     | score | rank
            ---|----------|-------|-----
            1  | Alice    | 95    | 1
            2  | Bob      | 90    | 2
            3  | Charlie  | 90    | 2
            4  | Dave     | 85    | 4
            

Prisma can sort — but it can’t calculate ranks by using a window function like RANK()  
So raw SQL is needed

Introduction to Prisma
----------------------

Modern ORM for Node.js and TypeScript

### Why Prisma?

*   Type-safe database client generated from schema
*   Intuitive data modeling with Prisma Schema
*   Auto-completion in your IDE
*   Powerful migration system
*   Great documentation and community
*   Works with PostgreSQL, MySQL, SQLite, SQL Server, MongoDB
*   Built-in connection pooling

### Installing Prisma

    # In your Express project
    cd backend
    
    # Install Prisma CLI as dev dependency
    npm install -D prisma
    
    # Install Prisma Client
    npm install @prisma/client
    
    # Initialize Prisma
    npx prisma init
    
    # This creates:
    # - prisma/schema.prisma (database schema)
    # - .env (with DATABASE_URL)

### Prisma Schema File

`prisma/schema.prisma` - Single source of truth

    // Database connection
    datasource db {
      provider = "postgresql"
      url      = env("DATABASE_URL")
    }
    
    // Prisma Client generator
    generator client {
      provider = "prisma-client-js"
    }
    
    // Your data models
    model User {
      id        Int      @id @default(autoincrement())
      email     String   @unique
      name      String?
      active    Boolean  @default(true)
      createdAt DateTime @default(now())
      updatedAt DateTime @updatedAt
    }

### Database Connection

Configure in `.env`

    # .env
    
    # PostgreSQL
    DATABASE_URL="postgresql://user:password@localhost:5432/mydb"
    
    # MySQL
    # DATABASE_URL="mysql://user:password@localhost:3306/mydb"
    
    # SQLite (for development)
    # DATABASE_URL="file:./dev.db"

### Prisma Workflow

1\. Define models in schema.prisma
   ↓
2. Run migration command
   npx prisma migrate dev --name init
   ↓
3. Prisma generates SQL migration files
   (stored in prisma/migrations/)
   ↓
4. Migration is applied to database
   ↓
5. Prisma Client is generated
   ↓
6. Use the type-safe Prisma Client in your code
    

### Running Migrations

    # Create and apply migration
    npx prisma migrate dev --name add_users_table
    
    # Apply migrations in production
    npx prisma migrate deploy
    
    # Reset database (WARNING: deletes all data)
    npx prisma migrate reset
    
    # View database in browser
    npx prisma studio

Migrations are version-controlled SQL files in `prisma/migrations/`

CRUD Operations with Prisma
---------------------------

### Create (INSERT)

    // Create single record
    const user = await prisma.user.create({
      data: {
        email: 'alice@example.com',
        name: 'Alice',
        active: true
      }
    });
    
    console.log(user);
    // { id: 1, email: 'alice@example.com', name: 'Alice', active: true, ... }
    
    // Create many records
    const users = await prisma.user.createMany({
      data: [
        { email: 'bob@example.com', name: 'Bob' },
        { email: 'charlie@example.com', name: 'Charlie' }
      ]
    });
    
    console.log(users.count); // 2

### Read (SELECT)

    // Find all
    const allUsers = await prisma.user.findMany();
    
    // Find one by unique field
    const user = await prisma.user.findUnique({
      where: { email: 'alice@example.com' }
    });
    
    // Find first match
    const firstActive = await prisma.user.findFirst({
      where: { active: true }
    });
    
    // Find or throw error
    const user = await prisma.user.findUniqueOrThrow({
      where: { id: 1 }
    });
    // Throws NotFoundError if not found

### Update

    // Update single record
    const updated = await prisma.user.update({
      where: { id: 1 },
      data: { name: 'Alice Smith' }
    });
    
    // Update many records
    const result = await prisma.user.updateMany({
      where: { active: false },
      data: { active: true }
    });
    
    console.log(result.count); // Number of updated records
    
    // Upsert (update or create)
    const user = await prisma.user.upsert({
      where: { email: 'new@example.com' },
      update: { name: 'Updated Name' },
      create: { email: 'new@example.com', name: 'New User' }
    });

### Delete

    // Delete single record
    const deleted = await prisma.user.delete({
      where: { id: 1 }
    });
    
    // Delete many records
    const result = await prisma.user.deleteMany({
      where: { active: false }
    });
    
    console.log(result.count); // Number of deleted records
    
    // Delete all records (careful!)
    await prisma.user.deleteMany({});

### Complete CRUD Example in Express

    import express from 'express';
    import prisma from './db.js';
    
    const app = express();
    app.use(express.json());
    
    // CREATE
    app.post('/api/users', async (req, res) => {
      const user = await prisma.user.create({
        data: req.body
      });
      res.json(user);
    });
    
    // READ ALL
    app.get('/api/users', async (req, res) => {
      const users = await prisma.user.findMany();
      res.json(users);
    });
    
    // READ ONE
    app.get('/api/users/:id', async (req, res) => {
      const user = await prisma.user.findUnique({
        where: { id: parseInt(req.params.id) }
      });
      res.json(user);
    });
    
    // UPDATE
    app.put('/api/users/:id', async (req, res) => {
      const user = await prisma.user.update({
        where: { id: parseInt(req.params.id) },
        data: req.body
      });
      res.json(user);
    });
    
    // DELETE
    app.delete('/api/users/:id', async (req, res) => {
      await prisma.user.delete({
        where: { id: parseInt(req.params.id) }
      });
      res.json({ message: 'User deleted' });
    });

Filtering & Advanced Queries
----------------------------

### Basic Filtering

    // Simple where
    const activeUsers = await prisma.user.findMany({
      where: { active: true }
    });
    
    // Multiple conditions (AND)
    const users = await prisma.user.findMany({
      where: {
        active: true,
        email: { contains: '@example.com' }
      }
    });
    
    // OR conditions
    const users = await prisma.user.findMany({
      where: {
        OR: [
          { email: { contains: 'gmail.com' } },
          { email: { contains: 'yahoo.com' } }
        ]
      }
    });
    
    // NOT conditions
    const users = await prisma.user.findMany({
      where: {
        NOT: { email: { contains: 'spam.com' } }
      }
    });

### Filter Operators

    // String filters
    const users = await prisma.user.findMany({
      where: {
        name: { startsWith: 'A' },
        email: { endsWith: '@example.com' },
        bio: { contains: 'developer' }
      }
    });
    
    // Numeric filters
    const products = await prisma.product.findMany({
      where: {
        price: { gte: 100, lte: 500 },  // between 100 and 500
        stock: { gt: 0 },                // greater than 0
        discount: { not: 0 }             // not equal to 0
      }
    });
    
    // Array filters (in / notIn)
    const users = await prisma.user.findMany({
      where: {
        id: { in: [1, 2, 3, 5, 8] },
        role: { notIn: ['BANNED', 'SUSPENDED'] }
      }
    });

### Date Filtering

    // Date comparisons
    const recentUsers = await prisma.user.findMany({
      where: {
        createdAt: {
          gte: new Date('2024-01-01'),
          lte: new Date('2024-12-31')
        }
      }
    });
    
    // Last 7 days
    const weekAgo = new Date();
    weekAgo.setDate(weekAgo.getDate() - 7);
    
    const recent = await prisma.user.findMany({
      where: {
        createdAt: { gte: weekAgo }
      }
    });

### Sorting & Pagination

    // Order by
    const users = await prisma.user.findMany({
      orderBy: { createdAt: 'desc' }
    });
    
    // Multiple sort fields
    const users = await prisma.user.findMany({
      orderBy: [
        { active: 'desc' },
        { name: 'asc' }
      ]
    });
    
    // Pagination
    const page = 2;
    const pageSize = 10;
    
    const users = await prisma.user.findMany({
      skip: (page - 1) * pageSize,
      take: pageSize,
      orderBy: { id: 'asc' }
    });
    
    // Get total count for pagination
    const total = await prisma.user.count();
    const totalPages = Math.ceil(total / pageSize);

### Select Specific Fields

    // Select only certain fields
    const users = await prisma.user.findMany({
      select: {
        id: true,
        name: true,
        email: true
        // password field will NOT be included
      }
    });
    
    // Result: [{ id: 1, name: 'Alice', email: 'alice@...' }, ...]
    
    // Exclude fields (alternative approach)
    const users = await prisma.user.findMany({
      omit: {
        password: true  // Available in Prisma 5.13+
      }
    });

### Aggregations

    // Count
    const userCount = await prisma.user.count();
    const activeCount = await prisma.user.count({
      where: { active: true }
    });
    
    // Aggregations
    const aggregations = await prisma.order.aggregate({
      _sum: { total: true },
      _avg: { total: true },
      _max: { total: true },
      _min: { total: true },
      _count: true
    });
    
    console.log(aggregations);
    // { _sum: { total: 10000 }, _avg: { total: 250 }, ... }
    
    // Group by
    const stats = await prisma.order.groupBy({
      by: ['status'],
      _count: { id: true },
      _sum: { total: true }
    });
    
    // Result: [
    //   { status: 'PENDING', _count: { id: 10 }, _sum: { total: 5000 } },
    //   { status: 'COMPLETED', _count: { id: 25 }, _sum: { total: 12500 } }
    // ]
