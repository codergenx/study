### Let's start with HTML!

HTML = Hyper Text Markup Language

But before that, do you know what's common between these pictures?

### Try it now

If you are using Chrome, open any webpage, right click and press "View Page Source".

You'll see similar code that we saw earlier.

Did you notice the text with the angle brackets?

These are called **tags**

Tags act like containers which let you know about the information enclosed within it.

Tags come in pairs (but not always) and are called **element**

### A closer look at tags

![](images/html-tag-opening.png) ![](images/html-tag-closing.png)

Additionally, tags can have additional information called **attributes**

![](images/tag-attribute.png)

They appear in the opening tag of an element

Attributes have a **name** and can be assigned a **value**

Inside the html element, there are two elements

*   **<body>**, which shows whatever is placed inside it in the browser
*   **<head>**, which holds the information about the page

**<title>** tag is present in the head element which shows the title you see on the browser tab for that page.

### Fun fact 😮

In the past, browsers didn't have tabs. There would be a separate window for every opened website.

Links in HTML
-------------

Links are created with the `<a>` tag (anchor tag).

They allow users to navigate between pages, sections, or even trigger actions like opening email.

Basic Link
----------

[Visit Example](https://www.example.com)

    <a href="https://www.example.com">Visit Example</a>

Open in New Tab
---------------

[Open Example in new tab](https://www.example.com)

    <a href="https://www.example.com" target="_blank">Open Example in new tab</a>

Email Links
-----------

[Send Email](mailto:someone@example.com)

    <a href="mailto:someone@example.com">Send Email</a>

Page Anchors
------------

[Jump to Section 2](#section2)

    <a href="#section2">Jump to Section 2</a>

**This is Section 2**

    <p id="section2">This is Section 2</p>

Image as a Link
---------------

[![Example](https://placehold.co/200)](https://www.example.com)

    <a href="https://www.example.com">
      <img src="https://placehold.co/200" alt="Example">
    </a>

Relative vs Absolute Links
--------------------------

**Relative Link:** Works within the same site.

[About Us](about.html)

    <a href="about.html">About Us</a>

**Absolute Link:** Full URL including domain.

[About Us](https://www.example.com/about.html)

    <a href="https://www.example.com/about.html">About Us</a>

From <div> to Semantic Containers
---------------------------------

In early HTML, developers used `<div>` everywhere as a general container.

HTML5 introduced **semantic elements** that give meaning to the structure.

### Using <div> Everywhere

Header

Navigation Links

Main Content

Footer

    <div id="header">Header</div>
    <div id="nav">Navigation Links</div>
    <div id="content">Main Content</div>
    <div id="footer">Footer</div>

### Semantic HTML5 Structure

Header

Navigation Links

Main Article Content

Footer

    <header>Header</header>
    <nav>Navigation Links</nav>
    <section>
      <article>Main Article Content</article>
    </section>
    <footer>Footer</footer>

### Why Semantic Containers?

**Accessibility** - screen readers understand page structure.

**SEO** - search engines recognize important content.

**Readability** - code is easier to maintain and understand.

We have not explored all the HTML elements. You can find more details regarding them online.
